## 救援工具

### fastboot_rescue.bat

**功能：** Fastboot模式救援和刷机工具

**使用场景：**
- 手机进入Fastboot模式无法启动
- 需要刷入系统镜像
- 需要恢复出厂设置

**使用方法：**
```batch
cd 工具脚本
.\fastboot_rescue.bat
```

**支持操作：**
1. 重启到正常模式
2. 重启到Recovery模式
3. 查看设备信息
4. 刷入recovery.img
5. 刷入boot.img
6. 刷入系统镜像
7. 擦除数据

**注意事项：**
- 需要设备处于Fastboot模式
- 刷机需要对应的镜像文件
- 擦除数据会清除所有用户数据

## 连接工具

### adb_stable_manager.bat

**功能：** ADB稳定连接管理器

**使用场景：**
- ADB连接不稳定
- 需要长时间保持连接
- 自动重连

**使用方法：**
```batch
cd 工具脚本
.\adb_stable_manager.bat
```

**功能特点：**
- ✅ 清理旧ADB进程
- ✅ 优化连接参数
- ✅ 心跳检测（每30秒）
- ✅ 自动重连

**关键优化：**
```batch
# 保持设备唤醒
settings put global stay_on_while_plugged_in 7

# 延长超时
settings put global adb_timeout 7200000
```

## 使用指南

### 场景1：首次部署

```bash
# 1. 连接手机并授权USB调试

# 2. 运行一键部署
.\deploy_all.bat

# 3. 在手机上打开Termux，运行：
termux-setup-storage
bash /sdcard/verify_termux.sh
```

---

### 场景2：设备救援

```bash
# 1. 确认设备处于Fastboot模式
.\工具软件\platform-tools\fastboot.exe devices

# 2. 运行救援工具
.\fastboot_rescue.bat

# 3. 选择操作（如重启）
# 输入: 1
```

---

### 场景3：ADB连接问题

```bash
# 1. 运行完整诊断
.\adb_full_check.bat

# 2. 如果检测到设备，运行连接管理器
.\adb_stable_manager.bat

# 3. 保持窗口开启以维持连接
```

---

### 场景4：使用AutoGLM

```bash
# 推荐方式：一键启动
.\autoglm_one_click.bat "打开设置查看设备信息"

# 或交互模式
.\autoglm_one_click.bat
# 输入任务描述

# 任务完成后选择Y继续下一个任务
```

---

### 场景5：连续执行任务

```bash
# 使用一键启动器
.\autoglm_one_click.bat "打开设置"
# 完成后输入Y

# 然后会提示输入下一个任务
```
